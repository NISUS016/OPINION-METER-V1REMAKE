# OpinionMeter — Design Document (v1)

## Overview

OpinionMeter is a sentiment analysis web app for online product reviews.
Users search for a product keyword, the app fetches matching reviews from
the Flipkart reviews dataset, runs them through an NLP + Logistic Regression
pipeline, and displays a sentiment breakdown chart alongside individual
review cards.

**Version:** v1 (submission build)
**Stack:** React (Vite) + FastAPI + scikit-learn
**Hosting:** Local demo first — Render.com if time permits

## Dataset

**Flipkart Product Reviews**

- Location: `flipkart-dataset/` — agent should glob the first CSV
  file found in this folder

| Column | Description |

| `Review` | Review body text — input to NLP pipeline |

## ML Pipeline

### Training — `train_model.py` (run once offline)

```
flipkart-dataset/*.csv
  → load columns: Review, Sentiment
  → normalize Sentiment to lowercase
  → drop nulls
  → text preprocessing via preprocess.py
  → TF-IDF vectorization
  → train 3 models, pick best by accuracy
  → save model.pkl, vectorizer.pkl, label_encoder.pkl → backend/models/
```

### Text Preprocessing — `backend/preprocess.py`

Apply in this exact order:

1. Lowercase
2. Strip HTML tags (`<[^>]+>`)
3. Strip URLs (`http\S+`)
4. Remove non-alphabetic characters
5. Tokenize by whitespace
6. Remove NLTK English stopwords
7. Porter stemming
8. Rejoin tokens

```python
import re
import nltk
from nltk.corpus import stopwords
from nltk.stem import PorterStemmer

nltk.download("stopwords", quiet=True)
stop_words = set(stopwords.words("english"))
stemmer = PorterStemmer()

def preprocess(text: str) -> str:
    text = str(text).lower()
    text = re.sub(r"<[^>]+>", " ", text)
    text = re.sub(r"http\S+|www\S+", " ", text)
    text = re.sub(r"[^a-z\s]", " ", text)
    tokens = text.split()
    tokens = [stemmer.stem(t) for t in tokens if t not in stop_words and len(t) > 2]
    return " ".join(tokens)
```

### TF-IDF Config

```python
TfidfVectorizer(
    max_features=50_000,
    ngram_range=(1, 2),   # bigrams catch "not good", "very bad"
    min_df=2,
    sublinear_tf=True
)
```

### Model

- Primary: **Logistic Regression** (`C=5`, `max_iter=1000`)
- Also train Naive Bayes and Linear SVM for comparison
- Save whichever scores highest accuracy on 20% test split
- Expected accuracy: ~88–92%

### Inference — `backend/predict.py`

```python
import pickle
from preprocess import preprocess

model     = pickle.load(open("models/model.pkl",      "rb"))
vectorizer = pickle.load(open("models/vectorizer.pkl", "rb"))

def predict(text: str) -> dict:
    clean = preprocess(text)
    vec   = vectorizer.transform([clean])
    label = model.predict(vec)[0]
    proba = model.predict_proba(vec).max()
    return {"label": label, "confidence": round(float(proba), 4)}
```

> Note: `predict_proba` is not available on LinearSVC. If LinearSVC wins,
> use `decision_function` and apply softmax, or fall back to Logistic
> Regression which always supports `predict_proba`.

## Backend — `backend/main.py`

**Framework:** FastAPI
**Run locally:** `uvicorn main:app --reload --port 8000`

### Startup

On app startup, load the dataset into a global pandas DataFrame in memory.
Do not reload per request.

```python
import glob, pandas as pd
from contextlib import asynccontextmanager
from fastapi import FastAPI

df = None

@asynccontextmanager
async def lifespan(app: FastAPI):
    global df
    files = glob.glob("flipkart-dataset/*.csv")
    df = pd.read_csv(files[0], usecols=["product_name", "Review", "Sentiment"])
    df["Sentiment"] = df["Sentiment"].str.strip().str.lower()
    df.dropna(subset=["Review", "Sentiment"], inplace=True)
    yield

app = FastAPI(lifespan=lifespan)
```

### CORS

Allow all origins for local development. Restrict to frontend URL in production.

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## API Endpoints

### `GET /health`

Health check.

**Response:**
```json
{ "status": "ok" }
```

---

### `GET /search?q={keyword}&limit={n}`

Search reviews by product keyword.

**Params:**
- `q` — search keyword (required)
- `limit` — max reviews to return (default: 50)

**Logic:** case-insensitive substring match on `product_name` column.
If fewer than 10 results found by product name, fall back to matching
on `Review` text instead.

**Response:**
```json
{
  "query": "phone",
  "count": 50,
  "reviews": [
    {
      "product_name": "Redmi Note 5 Pro",
      "text": "Battery life is amazing...",
      "sentiment": "positive"
    }
  ]
}
```

---

### `POST /analyze`

Run sentiment analysis on a list of review texts.

**Request body:**
```json
{
  "reviews": ["Great product!", "Terrible quality.", "It is okay."]
}
```

**Logic:** run each review through `predict.py` pipeline.

**Response:**
```json
{
  "summary": {
    "positive": 35,
    "neutral": 8,
    "negative": 7,
    "total": 50
  },
  "results": [
    { "text": "Great product!",     "label": "positive", "confidence": 0.95 },
    { "text": "Terrible quality.",  "label": "negative", "confidence": 0.91 },
    { "text": "It is okay.",        "label": "neutral",  "confidence": 0.74 }
  ]
}
```

## Frontend Components

### `App.jsx`

- Holds global state: `query`, `loading`, `results`, `summary`, `error`
- On search submit: calls `/search` then `/analyze` sequentially
- Renders `SearchBar`, and conditionally `ResultsChart` + list of `ReviewCard`

### `SearchBar.jsx`

- Controlled text input
- Submit on Enter or button click
- Shows loading spinner while requests are in flight
- Props: `onSearch(query)`, `loading`

### `ResultsChart.jsx`

- Renders a **pie chart** using [Recharts](https://recharts.org)
- Three segments: positive (green), neutral (amber), negative (red)
- Shows count + percentage label on each segment
- Props: `summary: { positive, neutral, negative, total }`

### `ReviewCard.jsx`

- Displays: truncated review text (max 200 chars), product name, sentiment badge
- Badge colors:
  - `positive` → green background
  - `neutral` → amber background
  - `negative` → red background
- Shows confidence as a percentage on the badge
- Props: `text`, `product_name`, `label`, `confidence`

## User Flow

```
1. User opens app → sees search bar centered on screen
2. Types a product keyword (e.g. "phone", "earphones", "shirt")
3. Hits Enter or clicks Search
4. Loading state shown
5. App calls GET /search?q=phone&limit=50
6. App calls POST /analyze with returned review texts
7. Results page renders:
   ├── Search query heading ("Results for: phone")
   ├── Overall sentiment badge (dominant sentiment)
   ├── Pie chart — positive / neutral / negative breakdown
   └── Scrollable list of ReviewCards
8. User can search again from the same bar
```
